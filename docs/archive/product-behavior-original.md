## PART 1: THE 3-TIER TRANSFORMATION SYSTEM

### Overview

```
TIER 1: COACH         →  TIER 2: SMART LENDER   →  TIER 3: WEALTH BUILDER
(Zero license)           (SAMA license)             (Partnerships)
Free                     Murabaha margins           Referral commissions
```

### Tier 1 — Coach (المستشار الذكي)

**What it does:**
- AI watches behavior. Advises. Prevents bad spending.
- Zero-friction tracking: voice, OCR, SMS, chat
- "Can I buy?" purchase simulation engine
- Commitment tracking (including BNPL installments)
- Savings goals with gap detection
- Integrity Score: how honest the user is with the system

**Gateway to Tier 2:**
User must pass behavioral test:
- 3+ months of consistent transaction logging
- Integrity Score above threshold (minimal gap between system data and user input)
- Spending patterns show improvement (declining impulse buys)
- At least 1 savings goal in progress

### Tier 2 — Smart Lender (المُقرِض الواعي)

**What it does:**
- Provides installment loans (Islamic Murabaha)
- Approval based on: BEHAVIOR + SALARY — not salary alone
- Dynamic credit ceiling: adjusts based on real-time spending behavior
- AI monitors: if behavior degrades, ceiling drops. If behavior improves, ceiling rises.
- Loan terms: better than Tabby/Tamara because risk is lower (behavioral screening)

**Gateway to Tier 3:**
- All bad debt repaid (consumption debt = zero)
- First savings goal achieved
- Surplus detected (consistent monthly excess after commitments)

### Tier 3 — Wealth Builder (باني الثروة)

**What it does:**
- AI detects investable surplus
- Routes to partner investment platforms (Alinma funds, Tamra, Abyan, Derayah)
- Tracks investment growth alongside spending
- Celebration milestone: "You were in debt 18 months ago. Now you have investments."

**Revenue:**
- Referral commission from investment platforms
- User stays free

## PART 2: THE 5-PHASE TRANSFORMATION JOURNEY

Based on behavioral economics: BJ Fogg (B=MAP), Nir Eyal (Hook Model), Teresa Amabile (Progress Principle).

### Phase 0 — The Wake-Up Call (الصدمة)
**Duration:** First 5 minutes of app use

| Behavior | How |
|----------|-----|
| User opens app first time | No registration. No forms. Just chat. |
| App asks 3 questions | Income, major commitments, approximate monthly spend |
| Shows instant insight | "You spend 73% of your income before mid-month. 7 out of 10 Saudis face the same. You're not alone." |
| Emotional impact | Recognition. Validation. "This app understands me." |
| Call to action | "خليني أساعدك. سجل أول عملية — بالصوت أو الصورة." |

### Phase 1 — Stabilization (الاستقرار)
**Duration:** Months 1-3

**Goal:** Stop the bleeding. Not build wealth — just stop the decline.

| Behavior | How |
|----------|-----|
| Silent Triage | Routine transactions (gas, utilities, normal groceries) — classified silently. No interruption. |
| Smart Intervention | Only intervene on "gray" or "red" transactions: unusual amounts, new categories, impulse patterns |
| Evening Check-in | ONE message at 9 PM: "3 عمليات تحتاج تصنيفها — متى ما فضيت اضغط المايك." |
| Small Wins | Weekly: "هذا الأسبوع وفرت ٢٠٠ ريال مقارنة بالأسبوع الماضي." |
| First Milestone | First month with remaining balance → celebration |
| No Investment Talk | Zero mention of investing. User is not ready. |

**User feeling:** "I am in control. For the first time, I know where my money goes."

### Phase 2 — Awareness (الوعي)
**Duration:** Months 3-6

**Goal:** Understand patterns. See the big picture.

| Behavior | How |
|----------|-----|
| Pattern Reveal | "You spend 40% of your income on restaurants and coffee." |
| Smart Alternatives | "If you buy a coffee machine once — you recover its cost in 7 weeks." |
| First Small Goal | "Let's save 500 SAR this month for an emergency fund." |
| Integrity Score Visible | User sees transparency metric — gamified progress |
| Social Proof | "Mohammed was 45K in debt. Now he saves monthly." |

**User feeling:** "I am beginning to understand myself. I see patterns I never noticed."

### Phase 3 — Liberation (التحرر)
**Duration:** Months 6-12

**Goal:** Exit bad debt. Build first savings.

| Behavior | How |
|----------|-----|
| Debt Avalanche | AI prioritizes high-interest debt first. Shows user the plan. |
| Debt Celebration | Every paid-off commitment: "خلصت من تابي! ٢٠٠ ريال فائض شهرياً 🎉" |
| Silver Tier | User promoted to "Silver" status in Azdal |
| Emergency Fund | First 1,000 SAR saved → milestone |
| Investment Introduction | First soft mention: "Did you know your 2,000 SAR surplus could earn 8% annually?" |

**User feeling:** "I am not that person anymore. I am progressing."

### Phase 4 — Growth (النمو)
**Duration:** Months 12-18

**Goal:** First investment.

| Behavior | How |
|----------|-----|
| Surplus Detection | AI confirms: "3 consecutive months with 1,500+ SAR surplus" |
| Investment Nudge | "Your money loses value sitting in checking. I found a fund matching your risk profile." |
| First Investment | 500 SAR — small but symbolic. User becomes an investor. |
| Gold Tier | User promoted to "Gold" status |
| Good Debt Only | Any debt remaining is asset-backed (mortgage, education, business) |

**User feeling:** "My money works for me now."

### Phase 5 — Freedom (الحرية)
**Duration:** Month 18+

**Goal:** Financial independence trajectory.

| Behavior | How |
|----------|-----|
| Auto-Invest | Surplus auto-routed to investments |
| Wealth Tracking | Net worth dashboard — not just spending |
| AI Role Shift | From "financial doctor" to "wealth coach" |
| Platinum Tier | User promoted to "Platinum" status |

**User feeling:** "I no longer need Azdal to survive. I use it to grow."

## PART 3: BEHAVIORAL UX — How We Keep Users Engaged

### The Silent Triage Protocol

| Transaction Type | Examples | AI Action |
|-----------------|----------|-----------|
| Green (Routine) | Gas 200, Electricity 350, Normal supermarket 300-500 | Auto-classify. Silent. No notification. |
| Gray (Ambiguous) | Jarir 800 SAR, Extra 1200 SAR | Flag for evening check-in. Frame as opportunity: "If this is educational → your score improves!" |
| Red (Impulse Alert) | Multiple purchases in 2 hours, late-night shopping, new store at high amount | Immediate notification: "يبدو أنك تشتري بشكل اندفاعي اليوم." |

### The Framing Effect

| Wrong Framing (Intrusive) | Right Framing (Empowering) |
|---------------------------|---------------------------|
| "تم خصم 800 ريال. الرجاء إدخال تفاصيل الفاتورة." | "بطل! عملية بـ 800 ريال. إذا كانت 'دورة تدريبية' — تراها ترفع مؤشر جدارتك. ودك تفصلها لي؟" |
| "أنت صرفت كثير هذا الأسبوع." | "مصاريفك هذا الأسبوع أعلى من المعتاد بـ 15%. تبي نشوف ليش؟" |
| "لا تشتري هذا." | "إذا اشتريت الآن → هدفك يتأخر 8 أشهر. إذا أجلت → توصل أسرع. الخيار لك." |

### The Hook Model Applied

| Hook Element | Azdal Implementation |
|-------------|---------------------|
| External Trigger | Bank transaction notification → Azdal asks for context |
| Action | User responds with voice (3 seconds) |
| Variable Reward | Integrity Score changes, tier progress updates, spending insight revealed |
| Investment | User adds goals, links accounts, builds history — sunk cost deepens |

### The Progress Principle

Users stay when they see progress. Not yearly progress — WEEKLY.

- Every week: "هذا الأسبوع أفضل من الأسبوع الماضي."
- Every debt paid: celebration animation
- Every goal milestone: tier badge upgrade
- Every month: "تقرير الوعي المالي" — personal newsletter

## PART 4: INTEGRITY SCORE — The Trust Metric

### What It Measures

| Factor | Weight | How Measured |
|--------|--------|-------------|
| Logging Consistency | 30% | Days with at least 1 transaction logged / total days |
| Receipt Upload Rate | 20% | Receipts uploaded vs gray-area transactions flagged |
| Data Match Accuracy | 25% | User input matches Open Banking ground truth |
| Response Time | 15% | How fast user responds to evening check-in |
| No Deletion Rate | 10% | Transactions not later deleted (editing is fine) |

### How It's Used

| Score Range | Effect |
|------------|--------|
| 90-100 | Platinum integrity. Full trust. Tier 2 gate unlocked. |
| 70-89 | Good integrity. On track. Tier 2 in progress. |
| 50-69 | Moderate. Gaps detected. Tier 2 delayed. |
| Below 50 | Low trust. Possible gaming. Tier 2 locked. |

### Anti-Gaming Design

- If user claims "50 SAR grocery" but Open Banking shows 500 SAR → Integrity penalty
- If user deletes transactions frequently → Integrity penalty
- If user logs only when reminded, never proactively → slow growth (not penalty)
- Integrity Score is NEVER the only factor for lending — always combined with real bank data

## PART 5: CORE BEHAVIORS (Existing — Preserved)

### 1. Compound Transaction Detection
User says "اشتريت بـ ٤٧٥ — قهوة وخضار ومطعم للعيال"
→ AI splits into 3 categorized transactions with shared group_id

### 2. Recurring Commitment Intelligence
"عندي تمارا ١٠٠٠ ياخذون ٢٠٠ كل شهر"
→ Tracks remaining, auto-includes in "Can I buy?", celebrates payoff

### 3. "Can I Buy?" — Purchase Decision Engine
Inputs: income + commitments + current spend + days to salary + active goals
Output: YES ✅ / WAIT ⚠️ / NO ❌ with explanation + alternative suggestions

### 4. Cold Start Intelligence
Never says "no data." Uses income brackets, general estimates, confidence levels. Gives value first, then asks minimal questions.

### 5. Gap Detection (Data vs Reality)
When user says "ما يبقى معي شيء" despite system data showing surplus: humble approach, 3 gap-filling questions, real recalculation.
